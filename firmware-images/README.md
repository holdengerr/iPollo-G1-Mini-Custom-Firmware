# Firmware Images

This directory is the inspectable, source-oriented view of what goes into public firmware images.

Use it when you want to understand:

- which profiles are bundled in a release
- which web/admin/miner features are expected in a release line
- what a given public image is supposed to contain

This directory is intentionally metadata-only.

It does **not** store:

- ready-to-flash release archives
- compressed release blobs
- vendor base images

Ready-to-flash artifacts belong in the GitHub Releases tab instead.
